sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types a note into the input field and clicks Save

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/notes
    activate server
    Note right of server: Server receives the new note data (JSON)
    Note right of server: Server stores the note in the database
    server-->>browser: 201 Created (or confirmation JSON)
    deactivate server

    Note right of browser: Browser updates the list of notes, optionally by fetching all notes again

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
    activate server
    server-->>browser: [{ "content": "HTML is easy", "date": "2023-1-1" }, { "content": "New note text", "date": "2026-2-26" }, ... ]
    deactivate server

    Note right of browser: Browser renders the updated list including the new note
